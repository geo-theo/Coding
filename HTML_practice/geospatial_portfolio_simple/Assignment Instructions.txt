Through these instructions below, you will build a grid/list of project cards, where clicking a card opens a modal with more information about that project.

Recommendations:
1. Keep it Concise: If you choose the modal path, avoid long-form case-study content. Aim for highlights, tech used, and links.
2. Don't Over-Scope; implement the assigned pattern cleanly before adding extra features.
3. Build in Order; HTML structure, then CSS, then JS behavior.
4. Semantic Element: Use the semantic <dialog> element. It has built-in functions available like my_dialog.showModal() and my_dialog.close()
5. Selecting Multiple Related Elements; Don't forget that .querySelectorAll() can be used to select each instance of a particular element based on selector (e.g., .chip or .card).

Part 1: Setup:
1. Create an index.html page in your new repo. It does not need a <header>, <footer>, but it must have the standard HTML page structure (<!doctype html>, <html>, <head>, <body>).
2. Create a Projects section with a heading (e.g., <h2>Projects</h2>). 
3. Create a project list/grid with at least 6 project cards that all share a common class (e.g., .project-card). 
4. Each project card must include a title (e.g., "Project 1") 
5. Each project card may optionally include (best to include more than just the title, try to make it look good): 
5.a. Short description (1-2 sentences or a few bullets) 
5.b. Preview image 
5.c. A category name (e.g., "Frontend", "UI/UX", "Games") 
6. Placeholder content is allowed: if you don't have 6 real projects yet, use placeholders. This project is graded on functionality and implementation quality, not content. 
7. Accessibility best practices: Do your best based on what we've covered (semantic elements, keyboard access, visible focus/hover states, color contrast ratios). 
8. Hover/focus states: add visible interaction feedback for clickable elements (cards, chips, links, close button, etc.).

Part 2: Card HTML Structure:
1. Use a real list for sets of cards: <ul>/<ol> with <li>. 
2. Use the semantic <article> element inside each list item. 
3. If the card triggers an action (modal), wrap the card in a button. If it navigates, use a link (see path-specific requirements for examples).

Part 3: Modal Implementation:
1. Each project card must open the modal.
2. Use one modal for the whole page, and swap its content based on which card was clicked.
3. Use an array of objects to store your project data and update the modal content from that data (e.g., projects[0], projects[1]).
4. Each project card button must have a data-index property that aligns with the index of its data in your array of objects (see HTML example below).
5. Modal content (placeholder content is okay):
5.a. Project title
5.b. Details (tools/tech, short description paragraph, etc.)
5.c. At least one link (e.g., repo, live, or placeholder)
6. The modal must have a close button inside it.
7. When the modal opens, background scrolling should be disabled. When the modal closes, background scrolling should be restored.

Optional Enhacement: Move focus to the modal when it opens using .focus() and tabindex="-1"
